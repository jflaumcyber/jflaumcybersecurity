Network Security — OSUIT Fall 2023

Penetration Testing
Report

A simulated attack on a Costco Wholesale network environment originating from a compromised end-user desktop. Conducted as a team pen test for Network Security 3443 at Oklahoma State University Institute of Technology.

Course: Network Security 3443
Date: Nov 2023
Team: Flaum, Youniacutt, Mang
Executive Summary

Attack Overview

This penetration test simulated a multi-vector attack against Costco's network infrastructure, targeting vulnerabilities in physical security, employee awareness, and web application defenses. The attack was scoped to demonstrate how an insider — or an actor with brief physical access — could compromise sensitive data including credentials, emails, and customer financial records.

The attack chain began with a USB Rubber Ducky drop at a publicly accessible employee desktop, captured session cookies via a man-in-the-middle phishing framework, and then pivoted laterally through the network using stolen credentials.

Attack Vectors

Methods Deployed

USB Rubber Ducky

Keystroke-injection device disguised as a USB drive. Plugged into an accessible employee desktop near the member service station — executes a pre-loaded script automatically.

Evilginx2 (MitM Phishing)

Reverse-proxy framework that intercepts real login sessions, capturing both credentials and session cookies — allowing bypass of 2FA (PingID) without needing the one-time code.

Spear Phishing

Targeted phishing emails impersonating IT or trusted management to trick higher-privilege employees into revealing credentials or downloading malware for privilege escalation.

SQL Injection

Automated scanning of web forms and API endpoints to inject malicious SQL queries, targeting Costco's e-commerce and internal application surfaces for database access.

Attack Chain

How the Breach Unfolded

The Rubber Ducky device was inserted into a USB port at a high-traffic kiosk near the store's restrooms — a location accessible to any passing member. Once plugged in, it silently launched Evilginx2, which operated as a transparent proxy between the employee and legitimate login pages.

When a supervisor accessed their email, Evilginx captured the authenticated session cookie and sent an alert to the attacker's device. The attacker injected the cookie using a browser extension, bypassing PingID 2FA entirely. From there, the compromised account enabled lateral movement: accessing shared drives, escalating privileges, and mapping the internal network for higher-value targets including the NAC server, email server, and wireless LAN controller.

Recommendations

Remediation Steps

  • Disable USB ports on all publicly accessible employee workstations or enforce strict device whitelisting via endpoint management.
  • Deploy advanced email security with anti-phishing headers and DMARC/DKIM enforcement to block impersonation attacks.
  • Implement hardware-based MFA (FIDO2/passkeys) resistant to session cookie hijacking, replacing SMS or app-based 2FA where possible.
  • Conduct regular SQL injection audits across all public-facing web forms and APIs; enforce parameterized queries in development standards.
  • Run recurring anti-phishing simulations and security awareness training — especially for employees with elevated access.
  • Establish a routine patch management cadence and periodic penetration testing program to identify new exposure points.
