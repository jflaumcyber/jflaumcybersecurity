Overview

Why Wireshark?

Wireshark is the industry-standard tool for network traffic analysis. It captures packets in real time (or reads from saved .pcap files), dissects them layer by layer, and lets you filter and follow individual conversations. Understanding what normal traffic looks like is the first step to recognizing what's wrong.

These labs cover the most common protocols encountered in both everyday networking and security investigations, building intuition for what traffic looks like at the packet level.

Lab Exercises

What Was Covered

TCP Three-Way Handshake

Capturing and labeling SYN, SYN-ACK, and ACK packets to visualize how connections are established and torn down.

HTTP Traffic Analysis

Following HTTP streams to reconstruct web requests and responses, including headers, cookies, and cleartext body content.

DNS Resolution

Tracing DNS queries and responses to understand how domain names are resolved and spotting potential DNS-based attacks.

TLS Handshake

Examining TLS negotiation packets to identify cipher suites, certificate exchanges, and the encrypted session setup.

Display Filters

Key Filter Syntax

Wireshark's display filter language is one of its most powerful features — it lets you zero in on exactly the traffic you need without touching the capture itself.

# Show only HTTP traffic http # Filter by IP address (source or destination) ip.addr == 192.168.1.1 # Show only DNS queries dns.flags.response == 0 # Follow a specific TCP stream tcp.stream eq 5 # Find packets with the SYN flag set tcp.flags.syn == 1 && tcp.flags.ack == 0 # Show traffic between two specific hosts ip.src == 10.0.0.1 && ip.dst == 10.0.0.2
Key Takeaway

The most important skill Wireshark teaches isn't operating the tool — it's building a mental model of what normal traffic looks like. Anomalies only stand out when you know the baseline.
