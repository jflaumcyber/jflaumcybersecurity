Concept

What Is Diffie-Hellman?

Diffie-Hellman (DH) is a key agreement protocol that allows two parties to independently derive the same shared secret, even if their entire exchange is observed by an eavesdropper. It works because of the mathematical difficulty of solving the discrete logarithm problem — computing the result of modular exponentiation is easy, but reversing it is computationally infeasible.

DH underpins modern TLS and is used every time you connect to an HTTPS website. The client and server negotiate a shared key without ever transmitting it directly — only public values cross the wire.

Step-by-Step Example

Using agreed public parameters modulus p = 5 and base g = 12, with private keys a = 3 (Jordan) and b = 6 (Professor Webber):

Public paramsp = 5, g = 12p = 5, g = 12
Private keya = 3b = 6
Compute public valueA = g^a mod pB = g^b mod p
Calculate12³ mod 5 = 1728 mod 5 = 312⁶ mod 5 = 2985984 mod 5 = 4
Exchange →Send A=3 to Webber · Send B=4 to Jordan
Compute secrets = B^a mod p = 4³ mod 5 = 4s = A^b mod p = 3⁶ mod 5 = 4

Both parties independently arrive at s = 4 — the shared secret. No eavesdropper watching the public exchange of A=3 and B=4 can derive the secret without solving the discrete logarithm.

OpenSSL Implementation

Running DH with OpenSSL

The same exchange was implemented using OpenSSL on Windows, generating real cryptographic DH parameters, per-party key pairs, and verifying that both derived secrets match at the byte level.

# Step 1 — Generate shared public parameters openssl genpkey -genparam -algorithm DH -out dhp.pem # View parameters (encoded and plain text) openssl pkeyparam -in dhp.pem -text # Step 2 — Each party generates their private/public key pair openssl genpkey -paramfile dhp.pem -out dhkey1.pem # Jordan openssl genpkey -paramfile dhp.pem -out dhkey2.pem # Webber # Step 3 — Extract public keys for exchange openssl pkey -in dhkey1.pem -pubout -out dhpub1.pem openssl pkey -in dhkey2.pem -pubout -out dhpub2.pem # Step 4 — Each party derives the shared secret openssl pkeyutl -derive -inkey dhkey1.pem -peerkey dhpub2.pem -out secret1.bin openssl pkeyutl -derive -inkey dhkey2.pem -peerkey dhpub1.pem -out secret2.bin # Step 5 — Verify both secrets are identical comp.exe secret1.bin secret2.bin

Both secret1.bin and secret2.bin produced the same hex output when viewed in a hex editor, confirming the key agreement worked correctly. This shared binary value is what would be used as a symmetric encryption key for further communication.
