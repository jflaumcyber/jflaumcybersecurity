Diffie-Hellman Key Exchange
OpenSSL · Modular Arithmetic · Applied Cryptography █
// overview

The Core Idea Intro to Applied Cryptography · OSUIT

Diffie-Hellman solves a deceptively hard problem: how do two parties agree on a shared secret over a completely insecure channel, without ever transmitting the secret itself? This lab works through the protocol manually with small numbers, then implements it with OpenSSL to verify the result.

The magic is in modular exponentiation — computationally easy to perform, computationally infeasible to reverse at large scales. That one-way property is what makes the exchange secure.

// manual walkthrough

Step-by-Step with Small Numbers

Agreed public parameters: p = 5 (prime modulus), g = 12 (generator/base). Private keys chosen independently: a = 3 (Jordan), b = 6 (Professor Webber).

StepJordanProfessor Webber
Compute public valueA = 12³ mod 5 = 3B = 12⁶ mod 5 = 4
Exchange public valuesSend A = 3 →← Send B = 4
Compute shared secrets = 4³ mod 5 = 4s = 3⁶ mod 5 = 4
Result: Both parties independently arrive at s = 4 — without ever transmitting the secret directly. An eavesdropper who saw A=3 and B=4 cannot recover the private keys or the shared secret without solving the discrete log problem.
// openssl implementation

Verified Key Agreement with OpenSSL

The manual walkthrough confirms the math. The OpenSSL implementation does the same with cryptographically secure parameters — and lets us verify that both sides derive an identical shared secret.

# 1. Generate shared public parameters (p and g) openssl genpkey -genparam -algorithm DH -out dhp.pem # 2. Each party generates their own key pair openssl genpkey -paramfile dhp.pem -out dhkey1.pem # Jordan openssl genpkey -paramfile dhp.pem -out dhkey2.pem # Webber # 3. Extract public keys to share openssl pkey -in dhkey1.pem -pubout -out dhpub1.pem openssl pkey -in dhkey2.pem -pubout -out dhpub2.pem # 4. Each party derives the shared secret using their private key + peer's public key openssl pkeyutl -derive -inkey dhkey1.pem -peerkey dhpub2.pem -out secret1.bin openssl pkeyutl -derive -inkey dhkey2.pem -peerkey dhpub1.pem -out secret2.bin # 5. Verify both secrets match comp.exe secret1.bin secret2.bin # → Files are identical ✓

Both secret1.bin and secret2.bin produced the same hex output, confirming successful key agreement without the secret ever crossing the network.
